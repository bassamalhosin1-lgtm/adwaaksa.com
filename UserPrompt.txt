رح أشرحلك منهجية واضحة + نقاط جاهزة تعطيها للعميل، لكن لازم تنفّذ أنت اختبارات السرعة و‑SEO بأدوات مثل PageSpeed / GTmetrix / Ahrefs لأن الوصول المباشر لموقعي العميل غير متاح من طرفي حالياً، وما في تقارير جاهزة عنهما على الويب. [gtmetrix](https://gtmetrix.com)

## 1. خطة التحليل اللي تعملها للموقعين

اعمل لكل موقع (cabltexperts.com و adwaaksa.com) نفس الباكدج:

1. سرعة وأداء (Performance / Core Web Vitals)  
   - اختبر الـ Home وأهم الصفحات بـ:
     - Google PageSpeed Insights (موبايل + دسكتوب).  
     - GTmetrix أو WebsiteAnalysis.site لأكثر تفاصيل كـ waterfall، حجم الصفحة، عدد الريكوست. [websiteanalysis](https://websiteanalysis.site)
   - سجّل:
     - LCP, CLS, TBT أو INP.  
     - حجم الـ HTML, CSS, JS, الصور.  
     - وقت الـ TTFB إن أمكن.  
   - اكتب ملاحظات: ملفات JS كثيرة؟ مكتبة ضخمة (Bootstrap/Theme جاهز)؟ صور غير مضغوطة؟

2. SEO تقني (Technical SEO)  
   استخدم أداة مثل SEOptimer / SEO Site Checkup / Alidahmash Analyzer لفحص أساسي. [seoptimer](https://www.seoptimer.com)
   راقب:
   - وجود ووضوح:
     - العنوان (title) + الوصف (meta description) لكل صفحة.  
     - هيكل العناوين H1–H6 منظم أو عشوائي.  
   - ملف robots.txt و sitemap.xml.  
   - Canonical tags.  
   - Structured data (Schema: LocalBusiness, Organization، منتجات لو في).  
   - HTTPS، عدم وجود mixed content.

3. SEO محتوى وكلمات مفتاحية (On‑Page & Keywords)  
   - حدد كلمات العميل الأساسية (مثلاً: “توريد كابلات كهربائية جدة”، “توريد إنارة الرياض”… حسب النشاط).  
   - افتح كل صفحة أساسية:
     - هل الـ H1 يحتوي الكلمة المستهدفة؟  
     - هل النص كافي (مش صفحة فاضية أو سلايدر فقط)؟  
     - هل في internal links بين الصفحات المهمة؟  
   - شوف الـ URL Slugs: مقروءة (مثلاً /services/lighting) أو عشوائية (/?p=123).

4. تجربة المستخدم والوصول (UX & Accessibility)  
   - تصميم موبايل: Responsive فعلياً أو مجرد تصغير الصفحة؟  
   - وضوح الـ Call To Action (اتصل بنا، اطلب عرض سعر).  
   - تباين الألوان (نص واضح على الخلفية)، حجم الخط.  
   - سهولة الوصول لبيانات التواصل والموقع على الخريطة.  
   - تحقق من وجود بدائل للنصوص في الصور alt text، وترتيب منطقي للعناصر لذوي الاحتياجات الخاصة.

5. قوة الدومين والروابط (Authority & Backlinks)  
   - استخدم أي أداة مجانية/حساب عندك: Ahrefs / Moz / Semrush (حتى لو Free):  
     - Domain Rating / Authority.  
     - عدد الروابط الخلفية + عدد الدومينات المحيلة.  
     - هل الروابط من مواقع حقيقية ولا سبام.  
   - شوف إذا في إدراج في Google Business Profile والدلائل المحلية، لأن هذا مهم للأعمال المحلية. [austinseo](https://austinseo.ai)

6. الأمان (Security)  
   - شهادة SSL صحيحة (HTTPS).  
   - عدم وجود تحذيرات browser.  
   - لو CMS مثل WordPress: تحقق من رأس X‑Powered‑By المكشوفة، نسخ قديمة، إلخ (بأدوات scan مجانية). [alidahmash](https://www.alidahmash.com/website-analyzer)

## 2. ملاحظات خاصة بإعادة البناء بـ HTML / تقنيات خفيفة

بما إن العميل طلب “لغات سريعة مثل HTML”، فكرة ممتازة خصوصاً لو الموقع عبارة عن بروفايل شركة + خدمات بدون Dashboard ضخم.

نصائح تقنية مباشرة لبناء نسخة محسّنة:

عبد الغني الحمدي, [26/02/2026 11:14 PM]
- ستاك مقترح  
  - HTML + CSS (Tailwind أو CSS خفيف من الصفر).  
  - JavaScript فقط للـ interactivity الضرورية (بدون jQuery أو plugins ثقيلة).  
  - استخدام Static Site (حتى لو تستعمل Generator مثل Astro / Next static export ثم ترفع الناتج كـ HTML).  
- أداء  
  - تصميم بدون سلايدر ثقيل ولا أنيميشن كثيرة.  
  - تحميل الصور:
    - WebP/AVIF إن أمكن.  
    - lazy‑loading للصور أسفل الـ fold.  
  - دمج CSS الضروري (Critical CSS) داخل <style> في الـ head لبداية أسرع. [austinseo](https://austinseo.ai)
  - تقليل أو إزالة أي JS غير ضروري، وإلغاء مكتبات UI العملاقة.
- SEO  
  - بناء معماري واضح:
    - / الرئيسية  
    - /about  
    - /services/lighting …  
    - /projects  
    - /contact  
  - لكل صفحة:
    - title فريد + meta description جذاب.  
    - H1 واحدة واضحة.  
    - فقرة تعريفية فيها الكلمات المفتاحية المستهدفة لكن بصياغة طبيعية.  
  - إضافة Schema LocalBusiness (اسم الشركة، الهاتف، العنوان، geo، أوقات العمل).  
- إدارة المحتوى  
  - لو العميل يحتاج تعديل نصوص باستمرار، ممكن:
    - يا Static + Panel بسيط (Headless CMS مثل Netlify CMS / Strapi).  
    - أو WordPress لكن مع:
      - Theme خفيف جداً.  
      - تعطيل كل الـ plugins غير الضرورية.  
      - استخدام caching قوي + CDN.  
- الوصول (Accessibility)  
  - عناوين منسقة H1–H2–H3.  
  - alt لكل صورة مهمة.  
  - أزرار وروابط واضحة (نصوص واضحة بدل “اضغط هنا”).

## 3. شغل تسليمه للعميل (Report Template)

اعمل ملف PDF أو Notion page لكل موقع، يتضمن:

| المحور | وضع الموقع الحالي | المشاكل | المقترحات عند إعادة البناء |
|--------|-------------------|---------|-----------------------------|
| السرعة | نتائج PageSpeed (مثلاً 45 موبايل، 80 دسكتوب) | حجم صفحة كبير، صور غير مضغوطة، JS كثير | تحويله لموقع static، ضغط الصور، تقليل JS، تفعيل cache و CDN. [gtmetrix](https://gtmetrix.com) |
| SEO تقني | لا يوجد sitemap، titles مكررة… | فهرسة ضعيفة، صعوبة في الترتيب | إنشاء sitemap.xml، ضبط titles, meta، هيكلة عناوين، إضافة Schema. [seoptimer](https://www.seoptimer.com) |
| المحتوى والكلمات | النص قليل أو غير موجه | ما في تركيز على Keywords محلية | كتابة محتوى موجه للكلمات الأساسية لكل خدمة / مدينة. |
| UX & موبايل | تصميم غير مريح/غير متجاوب تماماً | Bounce عالي محتمل | تصميم Responsive حقيقي، CTA واضحة، تبسيط التنقل. |
| الروابط والقوة | دومين Authority ضعيف، روابط قليلة | ظهور ضعيف في Google | بناء باك لينكات محلية، تفعيل Google Business، إدراج في أدلة شركات. |
| الأمان | HTTPS موجود/غير موجود | تحذيرات أو إعدادات سيئة | تفعيل SSL، إعداد HSTS، تحديثات إن كان CMS. |

تقدر تشتغل التحليل بنفسك خلال 1–2 ساعة لكل موقع مع الأدوات المجانية اللي فوق، وتطلع برپورت قوي يبرر إعادة البناء + يعطيك roadmap للتنفيذ.

لو حابب، ابعتلي لقطات شاشة من PageSpeed / GTmetrix + مثال من صفحة خدمة من كل موقع، وبجهزلك صياغة تقرير نهائي بالعربي تقدر تسلّمه للعميل مباشرة.
